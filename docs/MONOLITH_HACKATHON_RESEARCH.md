# MONOLITH Hackathon Research (Solana Mobile, 2026)

*Research conducted: March 8, 2026 (1 day before submission deadline)*

---

## 1. What is MONOLITH

**Official description:** "Build something they can't move." MONOLITH is the **second Solana Mobile Hackathon**, presented by **Solana Mobile** and **RadiantsDAO** (via their Align platform). It is a proving ground for builders bold enough to create what doesn't exist yet.

- **Organizers:** Solana Mobile + RadiantsDAO (with partners BONK and Helius)
- **Platform:** Align by Radiants (align.nexus)
- **Start date:** February 2, 2026
- **Submission deadline:** March 9, 2026 (00:00 UTC)
- **Duration:** 5 weeks
- **Participants:** 727 entrepreneurs registered; 143 submissions in the Mobile track (as of early March)
- **Tagline:** "Build something they can't move"

### Post-Submission Timeline
- **Voting starts:** March 10, 2026
- **Voting ends:** April 30, 2026
- **Prize distribution:** May 8, 2026

## 3. Tracks / Categories

There is effectively **one main track: Mobile**

- **Focus:** Mobile-first dApps for the Solana dApp Store
- **Open categories:** DeFi, payments, gaming, social, tools, utilities, experiments, or "entirely new categories"
- **Emphasis areas:** Usability, performance, and thoughtful mobile design that feels natural on a phone
- **Special interest (2026):** "Agentic" mobile apps -- tools where AI agents perform on-chain actions seamlessly in the background

### SKR Integration Bonus
Separate bonus prize for the best integration of SKR (Solana Mobile's Seeker token). This is an additional $10,000 in SKR tokens on top of the main prizes.

## 4. Judging Criteria

### Official Criteria (from the first Solana Mobile Hackathon -- likely same or similar for MONOLITH)

The evaluation panel assesses submissions on **five categories**:

1. **Completion** -- Does the demo show a functional, working application?
2. **Technical Depth** -- Code quality and complexity (verified via GitHub commits)
3. **Mobile Optimization** -- Effective use of mobile-specific features and UX
4. **Creative Solana Usage** -- Innovative blockchain integration approaches
5. **Vision and Clarity** -- How clearly the presentation communicates purpose and roadmap

### What Won in the First Hackathon (Patterns)
Winners demonstrated:
- Deep Solana blockchain integration
- Mobile-focused functionality that leverages device capabilities
- Effective use of the Solana Mobile Stack (SMS + Mobile Wallet Adapter)
- Real user traction or clear path to adoption

### What Colosseum/Solana Judges Generally Value
From the Colosseum submission guide:
- **Evidence of solving a real problem for real users** (this is the top signal)
- Product-market fit evidence (usage, community adoption, feedback)
- Clear market understanding and problem scale
- Strong founder potential and adaptability
- Technical depth and implementation reasoning
- Post-submission momentum (continuing to build after submission)

## 6. Submission Requirements

### Three Required Deliverables
1. **GitHub Repository** -- Complete, functional codebase
2. **Demo Video** -- Comprehensive showcase of app functionality (2-3 minutes, technical and direct)
3. **Pitch Deck** -- Vision presentation (templates available in Toolbox)

### Important Rules
- Apps must be built for **mobile** and utilize **Solana**
- You do **NOT** need to publish your app to the Solana dApp Store unless it's truly production-ready -- focus on a great demo
- Existing projects are permitted if they demonstrate **significant new mobile development during the hackathon period**
- Projects previously funded or that have won cash prizes in other hackathons are **ineligible**
- **AI-generated code or copied work = immediate disqualification**
- All code must be written by your team (open-source libraries are OK)
- All winners undergo **code verification** and may be asked follow-up technical questions
- **Zero tolerance** for false information -- forfeit all prizes

### Pitch Video Best Practices (from Colosseum guide)
- Maximum **3 minutes**
- Include team background and relevant experience
- Clearly explain: problem, solution, target audience
- Mention user validation or feedback received
- Approach like a startup pitch, not a product demo
- Focus on clear narrative over flashy production

### Technical Demo Best Practices
- Walk through core features built
- Explain the tech stack
- Detail design and implementation choices
- Clarify Solana integration and on-chain logic reasoning
- Show the interface and user flows

## 8. Solana Ecosystem Priorities (2026)

### What the Ecosystem Values Right Now
1. **Application-layer development** -- Moving beyond infrastructure to real-world use cases
2. **Mobile-first UX** -- Solana Mobile Seeker recorded $2.6B in volume in its first season
3. **Beyond memecoins** -- OG builders say "the next chapter is bigger than memecoins"
4. **TradFi on-chain** -- Bringing traditional finance onto Solana
5. **AI agents** -- Autonomous on-chain agents, especially on mobile (the "Agentic" narrative)
6. **DePIN** -- Decentralized physical infrastructure continues to be a priority
7. **Sustainable value capture** -- Not just speculation, but real economic activity
8. **Institutional readiness** -- Alpenglow consensus upgrade coming, predictable finality

### Mobile-Specific Ecosystem Signals
- Seeker device launching/expanding
- SKR token staking (nearly 70% of supply staked, 34,000 wallets)
- dApp Store as a genuine distribution channel (100+ apps)
- Solana Mobile providing workshops, documentation, and tools for builders
- Decal (stablecoin commerce + loyalty) highlighted as a standout app

## 9. Strategic Insights for Bondum

### How Bondum Aligns with What Judges Want
| Criteria | Bondum's Strength |
|---|---|
| **Real users** | PaniCafe partnership with ~8,000 real users |
| **On-chain transactions** | Real SPL token distribution, not simulated |
| **Mobile-first** | Built for Solana dApp Store, tested on Seeker |
| **Seed Vault support** | Tested on Saga Seeker -- real hardware integration |
| **Completion** | Live on Solana dApp Store (first version published) |
| **Mobile optimization** | Expo + NativeWind, native feel |
| **Creative Solana usage** | MWA + Privy, 2-step redemption, priority fees |
| **Consumer app** | Loyalty rewards = B2C, aligns with winner patterns |

### Potential Weaknesses to Address
- Not a game (gaming dominated past winners)
- Need a strong narrative: "loyalty infrastructure for Solana Mobile" not just "rewards app"
- Demo video and pitch quality matter enormously -- judges barely look at code unless you advance
- Must clearly show the PaniCafe traction as "proof this is real, not a hackathon throwaway"
- Consider the SKR integration bonus -- even a lightweight SKR integration could win extra $10K

### Key Competitor Signals from Past Winners
- **Decal** was highlighted as a standout Seeker app -- "stablecoin-native commerce with composable loyalty programs" -- this is VERY close to Bondum's space
- Bondum should differentiate: real partnership (PaniCafe), proven user adoption, on-chain proof
